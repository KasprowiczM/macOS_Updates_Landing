# macOS Updates Design System

Use this design system when creating or editing macOS Updates pages.

## Visual Direction

- Use blue as the brand accent: `#3278B8`, `#4AA3FF`, `#7BC4FF`.
- Use pale blue-grey surfaces in light mode and deep navy surfaces in dark mode.
- Keep CTAs, section labels, active states, icons, and proof highlights blue.
- Do not use legacy non-blue brand accents.

## Required Vocabulary

- Prescan
- Local inventory
- Installed software only
- System + Store
- CLI/npm + Homebrew
- Internet app handlers
- Postupdate history
- Private dev_sync overlay

## Avoid

- Maintenance-cleaner metaphors.
- Risk-framing language from unrelated utility products.
- Claims that missing apps are installed automatically.
- Any design that drifts away from the blue update-orbit identity.
